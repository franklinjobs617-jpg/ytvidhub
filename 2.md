# YTVidHub 工具页与 Workspace 分工方案

## 1. 这份文档解决什么问题

这份文档只回答一个问题：

> YTVidHub 已经有 `workspace` 了，为什么还需要 SEO 工具页？工具页应该怎么做？workspace 又该承担什么？

核心结论：

```text
工具页负责拿搜索流量
workspace 负责承接连续操作、登录、保存、付费
```

所以正确路径不是：

```text
Google 搜索 -> workspace
```

而是：

```text
Google 搜索 -> 工具页 -> 成功结果 -> 下一步动作 -> workspace
```

---

## 2. 为什么有了 workspace 还要做工具页

### 2.1 因为搜索引擎需要“明确任务页”

搜索引擎更容易理解这种页面：

- `youtube-subtitle-downloader`
- `youtube-transcript-generator`
- `youtube-subtitle-to-txt`
- `youtube-playlist-subtitle-downloader`

因为这些 URL、标题、H1、首屏任务都很清楚。

而 `workspace` 这种页面通常太宽：

- 功能多
- 任务多
- 入口多
- 目标多

Google 很难判断它到底最适合匹配哪个搜索词。

### 2.2 因为用户搜索时要的是“马上完成一个动作”

用户搜索：

- `youtube subtitle downloader`
- `download youtube subtitles`
- `youtube transcript generator`

这类词时，用户通常想的是：

> 我现在就要完成这一步。

所以 SEO 落地页必须让用户一进来就能操作，而不是先进一个复杂工作台。

### 2.3 因为 workspace 适合“后续流程”，不适合“首次获客”

workspace 更适合承接：

- 保存结果
- 编辑结果
- 批量处理
- 多步骤操作
- 历史记录
- 高级导出
- 登录与付费

所以它更像产品后台，而不是搜索落地页。

---

## 3. YTVidHub 的正确分工

## 3.1 工具页的职责

工具页负责：

1. 精准匹配一个搜索任务
2. 首屏直接让用户开始操作
3. 当前页展示结果
4. 结果后引导下一步动作
5. 把用户自然送进 workspace

一句话：

> 工具页解决“我现在就想做这一步”。

## 3.2 Workspace 的职责

workspace 负责：

1. 保存这次结果
2. 继续处理结果
3. 组合多个步骤
4. 历史记录与项目管理
5. 高级能力和付费承接

一句话：

> workspace 解决“这件事我还要继续做下去”。

---

## 4. 最推荐的用户路径

YTVidHub 最合理的路径应该是：

```text
用户搜索某个具体任务
-> 进入对应工具页
-> 直接完成当前任务
-> 在结果页看到下一步按钮
-> 点击进入 workspace
-> 登录 / 保存 / 批量 / 导出 / 付费
```

例如：

```text
Google 搜索：youtube subtitle downloader
-> /youtube-subtitle-downloader/
-> 成功下载字幕
-> 用户看到：转 TXT / 去时间戳 / 批量处理 / 打开 workspace
-> 进入 workspace 做更深操作
```

## 5. 页面层级应该怎么设计

建议结构如下：

```text
/
首页：品牌页 + 工具导航页

/youtube-subtitle-downloader/
工具页：单视频字幕下载

/youtube-transcript-generator/
工具页：转 transcript / 文本

/youtube-subtitle-to-txt/
工具页：字幕转纯文本 / 去时间戳

/youtube-playlist-subtitle-downloader/
工具页：playlist 或 channel 批量下载

/workspace
工作台：保存、编辑、批量、高级导出、登录、付费
```

---

## 6. 工具页应该怎么做

每个工具页只做一个主要任务，不要混。

标准结构建议如下：

1. `Title`
2. `Meta Description`
3. `H1`
4. 首屏工具区
5. 结果展示区
6. 下一步按钮区
7. 简短说明区
8. FAQ
9. 相关工具页和 workspace 链接

## 6.1 首屏工具区

必须满足：

- 用户一进来就能开始
- 不要求先登录
- 输入框和按钮明确
- 不让长文案挡住操作

例如：

```text
输入 YouTube URL
选择输出格式
点击 Download / Extract / Convert
```

## 6.2 结果展示区

原则：

- 结果在当前页展示
- 不要把结果藏太深
- 成功后用户能立刻感知到“我拿到了”

## 6.3 下一步按钮区

这是最关键的一块。

结果出来后，必须立刻给下一步动作，而不是只放说明文字。

建议按钮类型：

- `下载结果`
- `转成 TXT`
- `去除时间戳`
- `批量处理`
- `打开 workspace`

原则：

> 工具页不能在“结果出来”这一步结束，结果后面必须有下一步。

---

## 7. Workspace 应该怎么做

workspace 不再承担“SEO 精准匹配”的任务，它承担“持续使用”的任务。

建议最少包含这些模块：

1. 当前结果面板
2. 格式切换
3. 清洗选项
4. 导出功能
5. 历史记录
6. 项目保存
7. 批量任务入口
8. 登录 / 升级入口

### 7.1 workspace 该放哪些能力

建议放在 workspace：

- 保存历史
- 登录后继续处理
- 批量下载
- 多步骤编辑
- 项目管理
- 高级导出
- 多语言扩展
- 团队协作

### 7.2 哪些能力不该只放 workspace

这些能力如果是核心搜索意图，就不能只藏在 workspace：

- 单视频下载字幕
- 单视频生成 transcript
- 字幕转 TXT
- playlist 字幕批量下载

这些应该在工具页上就能直接做。

---

## 8. 工具页和 workspace 的边界

这是最重要的边界：

### 工具页

- 解决单步任务
- 直接拿到结果
- 低门槛开始
- 面向搜索用户

### workspace

- 解决连续任务
- 保存和再次编辑
- 面向已使用用户
- 适合登录、留存、付费

可以理解成：

```text
工具页 = 单步成功
workspace = 连续成功
```

---

## 9. 不要怎么做

### 9.1 不要把所有功能都塞进 workspace

问题：

- SEO 很难起量
- 搜索用户找不到立刻可用的入口
- `tool_start` 会很低

### 9.2 不要让工具页只剩介绍页

问题：

- 用户搜工具词进来，却不能直接操作
- 点击有了，使用没有

### 9.3 不要把工具页都做成完整 mini app 彼此孤立

问题：

- 用户做完一步就走
- 站内点击弱
- workspace 没有承接价值

### 9.4 不要让工具页和 workspace 完全重复

问题：

- 内部角色混乱
- 页面意图重叠
- SEO 和转化都会混乱

---

## 10. YTVidHub 当前最适合的页面分工

建议你先把站点收成这 4 个核心工具页 + 1 个 workspace：

### 核心工具页 1

`/youtube-subtitle-downloader/`

职责：

- 单视频字幕下载

首屏目标：

- 输入 URL
- 选择 `SRT / VTT / TXT`
- 立即下载

下一步按钮：

- `转成 TXT`
- `提取 transcript`
- `批量下载 playlist`
- `打开 workspace`

### 核心工具页 2

`/youtube-transcript-generator/`

职责：

- 获取 transcript / 文本

首屏目标：

- 输入 URL
- 生成 transcript

下一步按钮：

- `去除时间戳`
- `导出 TXT`
- `批量提取`
- `打开 workspace`

### 核心工具页 3

`/youtube-subtitle-to-txt/`

职责：

- 字幕转纯文本
- 去除时间戳

首屏目标：

- 输入 URL 或字幕内容
- 输出 clean text

下一步按钮：

- `下载 TXT`
- `提取更多 transcript`
- `批量处理`
- `打开 workspace`

### 核心工具页 4

`/youtube-playlist-subtitle-downloader/`

职责：

- playlist / channel 批量下载

首屏目标：

- 输入 playlist 或 channel URL
- 批量导出

下一步按钮：

- `导出 ZIP`
- `转 clean text`
- `保存到 workspace`
- `升级批量额度`

### Workspace

`/workspace`

职责：

- 保存结果
- 再编辑
- 批量
- 历史
- 项目
- 付费

## 11. 后续扩展页怎么长

当 4 个核心页稳定后，再扩第二层工具页。

第二层页建议只扩“下一步任务页”，不要扩同义词页。

适合扩的方向：

- `youtube-subtitle-to-srt`
- `youtube-subtitle-to-vtt`
- `youtube-transcript-without-timestamps`
- `youtube-channel-subtitle-downloader`
- `youtube-transcript-json-export`
- `download-youtube-captions-as-text`

不建议优先扩的方向：

- 只是把 `downloader / extractor / generator` 换着写
- 功能差不多但标题不同的页面

原则：

> 扩任务链，不扩近义词链。

---

## 12. 最终基调

YTVidHub 的页面基调建议固定成：

> `YTVidHub = YouTube subtitle and transcript workflow tools`

然后角色明确：

- 首页：工具集合与导航
- 工具页：解决单步任务
- workspace：承接连续任务和付费

如果只记一句话，就记这个：

> 工具页解决“我现在就想做这一步”，workspace 解决“我接下来还要继续处理这件事”。
