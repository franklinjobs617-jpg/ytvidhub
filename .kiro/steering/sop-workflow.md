# 工作流程偏好：SOP 优先

## 核心原则
用户偏好：**每次执行任务时，优先创建或参考 SOP 文档**

## 执行规则

### 当用户提出任务时，你必须：

1. **评估任务复杂度**
   - 是否需要多个步骤？
   - 是否会重复执行？
   - 是否容易出错？

2. **检查现有 SOP**
   - 查看 `docs/` 目录下是否有相关 SOP
   - 如果有，引用并按照执行

3. **创建新 SOP（如需要）**
   - 使用 `docs/SOP_TEMPLATE.md` 作为模板
   - 包含：前置条件、快速步骤、详细步骤、验证清单
   - 先创建 SOP，再执行任务

4. **执行并验证**
   - 严格按照 SOP 步骤执行
   - 完成后进行验证
   - 记录改进点

## 需要 SOP 的任务类型

🔴 **必须有 SOP**：
- 多语言发布
- 生产部署
- 数据库操作
- 安全相关操作

🟡 **建议有 SOP**：
- 功能开发流程
- 测试流程
- Bug 修复流程

## 现有 SOP
- `docs/MULTILINGUAL_SOP.md` - 多语言上线流程
- `docs/SOP_TEMPLATE.md` - SOP 创建模板

用户可以说：

- "创建 SOP"
